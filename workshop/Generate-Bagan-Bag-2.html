State Diagram: Siklus Hidup Work Order

Membuat diagram status untuk menggambarkan siklus hidup work order pemeliharaan.

Tolong buatkan state diagram untuk siklus hidup work order menggunakan Mermaid dengan spesifikasi berikut: States: - Draft (initial state) - Submitted - Approved - In Progress - On Hold - Completed - Closed - Rejected - Cancelled Transitions: - Draft -> Submitted: setelah input lengkap - Submitted -> Approved/Rejected: review supervisor - Approved -> In Progress: mulai pengerjaan - In Progress -> On Hold: menunggu part/approval - In Progress -> Completed: pekerjaan selesai - Completed -> Closed: verifikasi final - Any State -> Cancelled: pembatalan Tambahan: - Notes untuk setiap state - Composite states jika ada sub-status - Parallel states untuk concurrent activities - Entry dan exit actions Gunakan format: stateDiagram-v2 dengan: - State descriptions - Transition conditions - Nested states where applicable - Concurrent states if needed

Struktur Prompt:

  1. Definisi semua states
  2. Spesifikasi transitions
  3. Kondisi khusus (composite/parallel states)
  4. Actions dan events
  5. Notes dan descriptions
Sesuaikan dengan workflow approval dan prosedur dokumentasi di organisasi Anda.
2

ERD: Sistem Manajemen Aset Pembangkit

Membuat Entity Relationship Diagram untuk sistem manajemen aset yang akan dikembangkan.

Tolong buatkan ERD untuk sistem manajemen aset pembangkit menggunakan Mermaid dengan spesifikasi berikut: Entities: 1. Asset - AssetID (PK) - Name - Type - Location - InstallationDate - Status 2. Maintenance - MaintenanceID (PK) - AssetID (FK) - Type - ScheduledDate - Status 3. Spare_Parts - PartID (PK) - Name - Specification - Stock - MinimumStock 4. Work_Orders - WOID (PK) - AssetID (FK) - MaintenanceID (FK) - Status - Priority 5. Personnel - PersonnelID (PK) - Name - Role - Certification Relationships: - Asset 1--* Maintenance - Asset 1--* Work_Orders - Maintenance 1--1 Work_Orders - Spare_Parts *--* Work_Orders - Personnel *--* Work_Orders Gunakan format: erDiagram dengan: - Proper relationship notations - Attribute lists - Key indicators - Relationship descriptions

Struktur Prompt:

  1. Definisi entitas dengan atribut
  2. Spesifikasi primary dan foreign keys
  3. Definisi relationships
  4. Kardinalitas relationships
  5. Deskripsi tambahan bila diperlukan
Sesuaikan dengan kebutuhan sistem dan tambahkan entitas spesifik yang relevan dengan operasional pembangkit Anda.
