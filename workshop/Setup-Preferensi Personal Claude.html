Preferensi personal membantu Claude memahami kebutuhan spesifik Anda dan memberikan respons yang lebih relevan dalam konteks industri pembangkit listrik.

1. Identitas & Peran

Anda dapat memanggil saya [nama]. Saya bekerja di [unit pembangkit/divisi].
Peran saya mencakup [deskripsi peran], fokus pada [area spesifik].

Contoh:

  • Saya Budi dari Operasi PLTU Suralaya Unit 4-7
  • Saya Sarah dari Divisi Supply Chain Management
  • Saya Darto, Petugas K3 di PLTGU Gresik

2. Preferensi Teknis & Bahasa

Preferensi teknis:
- Standar teknis: [ASME/IEEE/IEC/SNI/dll]
- Sistem satuan: [Metrik/Imperial]
- Format dokumentasi: [format standar perusahaan]

Preferensi bahasa:
- Bahasa utama: [Indonesia/Inggris]
- Istilah teknis: Selalu dalam Bahasa Inggris
- Dokumentasi: Bilingual jika diperlukan

3. Konteks Pekerjaan

Konteks pekerjaan:
- Sistem utama: [sistem/peralatan utama]
- Proses kunci: [proses-proses kunci]
- Tugas rutin: [tugas-tugas rutin]
- Tantangan umum: [tantangan umum]

Contoh untuk Operator:

Konteks pekerjaan:
- Sistem utama: Turbin Gas GE 9FA, HRSG, Turbin Uap
- Proses kunci: Start-up, shutdown, perubahan beban
- Tugas rutin: Monitoring parameter, pengisian log sheet
- Tantangan umum: Load rejection, penanganan trip

4. Format Output yang Diinginkan

Preferensi output:
- Gaya dokumentasi: [format standar perusahaan]
- Penyajian data: [tabel/grafik/dashboard]
- Struktur laporan: [sesuai template perusahaan]
- Format file: [Excel/PDF/Word/dll]

5. Preferensi Kolaborasi & Komunikasi

Preferensi kolaborasi:
- Format rapat: [online/offline/hybrid]
- Saluran komunikasi: [email/WhatsApp/Teams/dll]
- Urgensi respons: [normal/urgent/critical]
- Preferensi notifikasi: [real-time/rangkuman harian/mingguan]

6. Inovasi & Improvement

Preferensi inovasi:
- Pendekatan problem solving: [metode analisis]
- Framework improvement: [Kaizen/Six Sigma/PDCA/dll]
- Format dokumentasi: [A3 Report/DMAIC/dll]
- Metrik keberhasilan: [KPI/OPI yang relevan]

Panduan Penggunaan

Cara Mengatur Preferensi:

  1. Buka Settings di Claude
  2. Pilih bagian "What preferences should Claude consider in responses?"
  3. Masukkan preferensi sesuai template di atas
  4. Sesuaikan dengan peran dan kebutuhan spesifik

Tips Penggunaan:

  • Mulai dengan preferensi dasar, tambahkan detail sesuai kebutuhan
  • Gunakan bahasa yang jelas dan spesifik
  • Update preferensi saat ada perubahan signifikan
  • Kombinasikan dengan Project Instructions untuk hasil optimal

Contoh Penerapan:

Untuk Tim Operasi:

Anda dapat memanggil saya Budi. Saya Operator Senior di PLTU Unit 5.
Saya bekerja dengan sistem DCS Ovation dan menangani operasi harian.
Ketika mendiskusikan masalah teknis:
- Gunakan standar ASME untuk referensi
- Cantumkan parameter kritis dalam kedua satuan (KPA & PSI)
- Sertakan referensi P&ID terkait
- Prioritaskan pertimbangan keselamatan
Format output dalam format logsheet standar jika diperlukan.

Untuk Tim Supply Chain:

Anda dapat memanggil saya Sarah dari Procurement.
Saya menangani pengadaan spare parts untuk sistem turbin.
Preferensi:
- Gunakan kode material SAP
- Sertakan lead time & kritikalitas stok
- Referensikan SOP pengadaan terkait
- Format dalam template PR/PO standar
Tonjolkan persyaratan kepatuhan yang ada.
